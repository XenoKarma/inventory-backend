🔥 E. STOCK MANAGEMENT (Core System)
Stock Movements (Log semua perubahan stok)
Method	Endpoint	Keterangan
GET	/api/stock-movements?page=1&product_id=1&warehouse_id=1&type=in	List movements (filterable)
POST	/api/stock-movements	Create movement (auto update stock)
GET	/api/stock-movements/{id}	Show movement detail
Request Body (POST):

// STOCK IN
{
  "type": "in",
  "product_id": 1,
  "to_warehouse_id": 1,
  "quantity": 100,
  "note": "Barang masuk dari supplier"
}

// STOCK OUT
{
  "type": "out",
  "product_id": 1,
  "from_warehouse_id": 1,
  "quantity": 10,
  "note": "Barang keluar untuk dijual"
}

// TRANSFER (Antar Gudang)
{
  "type": "transfer",
  "product_id": 1,
  "from_warehouse_id": 1,
  "to_warehouse_id": 2,
  "quantity": 50,
  "note": "Pindah stok ke gudang lain"
}
⚠️ RULES:

Stock In: Wajib to_warehouse_id, product_id, quantity
Stock Out: Wajib from_warehouse_id, product_id, quantity (cek stok cukup)
Transfer: Wajib from_warehouse_id & to_warehouse_id (cek stok cukup)
User ID: Otomatis dari auth
Database Transaction: Konsistensi data terjamin
